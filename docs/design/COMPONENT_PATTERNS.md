# أنماط المكوّنات — تدفّق الخير

> جرد للمكوّنات الموجودة فعلياً ونيّتها التصميمية وقواعد إعادة استخدامها.
> الأسماء تطابق أصناف CSS الفعلية ليسهل الربط بالكود.

---

## 1. الشريط العلوي (`.f-top`)
- **المادة:** زجاج مصنفر `backdrop-filter: blur(22–28px) saturate(1.8)`، حدّ سفلي `0.5px`، sticky.
- **المكوّنات:** علامة (`.f-brand`) + فاصل مائل + شارة الفريق (`.f-team`) كبسولة زجاجية + بحث متحوّل + أزرار أيقونية دائرية + Avatar بقائمة منسدلة.
- **القاعدة:** يبقى دائماً فوق المحتوى؛ الزجاج يكشف ما تحته بهدوء.

## 2. التبويبات (`.f-tabs` / `.f-tab`)
- **النمط:** Segmented Control بنمط iOS — كبسولة عائمة، التبويب النشط له سطح أبيض + ظل خفيف، الفقاعة تتبع بزنبرك.
- **القاعدة:** التبويب النشط واحد فقط؛ الأيقونة تمتلئ (FILL 0→1) عند التنشيط.

## 3. البحث المتحوّل (`.f-search` — نمط MORPH)
- **السلوك:** دائرة `36px` صامتة → تتمدّد لحقل كامل `clamp(280px, 32vw, 420px)` عند hover/focus، كاشفةً placeholder و`⌘K`.
- **يخدم:** progressive-disclosure — لا ضجيج دائم.

## 4. بطاقات الإحصاء (`.f-stat`)
- **التركيب:** تسمية + أيقونة ملوّنة بالقسم + دلتا (up أخضر / dn أحمر) + رقم كبير (Geist، تتبّع سالب) + وحدة + sub بخط mono + **sparkline** (`.f-spark`).
- **التفاعل:** رفع `translateY(-3px)` + ظل أعمق + امتلاء الأيقونة عند hover.
- **القاعدة:** الأرقام جدولية؛ التلوين خافت (radial 6%).

## 5. بطاقات الأقسام (`.f-dept`) + لوحة الأقسام المتوسّعة (`.f-deck`)
- **`.f-dept`:** أيقونة بتدرّج القسم + كود mono + اسم + وصف + إحصاءات بفاصل منقّط + chevron يتحرّك. خلفية radial pattern تظهر عند hover.
- **`.f-deck__panel`:** لوحة تتمدّد `flex: 1 → 2.7` بزنبرك `620ms`؛ علامة مائية ضخمة للكود (`5.4rem`)؛ حلقة طاقة (energy ring) conic-gradient مصنفرة؛ المحتوى المصغّر يتلاشى والكامل يظهر بتأخير.
- **القاعدة:** لوحة مفتوحة واحدة في كل مرة؛ مؤشّرات نقاط أسفلها.

## 6. بطاقة Spotlight (`.f-spot`)
- **النمط:** بطاقة مميّزة **داكنة** بتدرّج لون القسم + هالات radial بيضاء/ذهبية + أيقونة زجاجية `84px` + chips mono + CTA أبيض + شريط تقدّم + نقاط + دوران تلقائي قابل للإيقاف.
- **القاعدة:** عنصر توقيع واحد للصفحة؛ لا تُكرّر بكثرة.

## 7. صفوف الخدمات السريعة (`.f-exp` — express)
- صف مدمج: أيقونة ملوّنة بالقسم + اسم + meta mono + سهم يتحرّك. شبكة 4→2→1.

## 8. تغذية النشاط (`.f-feed`)
- صفوف: نقطة قيادة ملوّنة بالحالة + عنوان + كود ملوّن + sub + حالة بنقطة + وقت mono + chevron. صفّ كامل قابل للنقر، hover يلوّن الخلفية.

## 9. أشرطة الاستخدام (`.f-usage`)
- تسمية + كود mono + رقم mono + شريط `5px` بتعبئة متدرّجة (لون → ذهب). انتقال العرض `520ms`.

## 10. بطاقات التذكير (`.f-rem`)
- شريط متدرّج علوي `3px` (لون → ذهب/عنّاب) + وسم ملوّن + عنوان + نص + تذييل منقّط + CTA ملوّن.
- **حالة الإضافة (`.f-rem--add`):** حدّ متقطّع، مُعطّلة، وسم «قريباً» ذهبي → نمط **placeholder/empty مهذّب**.

## 11. التذكرة الرقمية (`.hub-pass` — wallet pass)
- بطاقة تميل في 3D وتتبع المؤشّر (fine pointer فقط)، أيقونة ترتفع وتميل عند التركيز. لمسة فاخرة.

## 12. شريط النصائح الدوّار (`.f-ticker`)
- شريط: قضيب تقدّم رأسي + أيقونة ملوّنة + وسم + عنوان + نص (سطران clamp) + تنقّل + عدّاد. دوران رأسي للمحتوى.

## 13. النماذج (Forms)
- **التبويبات (`.ff-tabs`):** segmented زجاجي (نمط G2).
- **علامة الصح (`.ff-check`):** مرسومة بزنبرك مع overshoot (drawn checkmark).
- **الخيارات المجزّأة (`.ff-seg__opt`):** تنبض عند الاختيار.
- **الحقول:** حدّ `0.5px`، حلقة تركيز iOS تنمو، ink داخلي خفيف.
- **لوحة الرسوم (`.ff-feepanel`):** الإجمالي «يتنفّس» (لون) عند التغيّر؛ أرقام جدولية.
- **الحفظ التلقائي (`.ff-autosave__dot`):** نقطة تتنفّس باستمرار → transparency + mixed-initiative.
- **الورقة (`.ff-paper`):** ترتفع قليلاً عند hover كرفع صفحة.

## 14. تفاصيل الحالة (Cases) + نافذة التراجع (`.undo-win`)
- صفوف حالات بأولوية (standard/vip/urgent) وحالة نصّية؛ نقطة القيادة تكبر عند hover؛ المراحل (timeline) تظهر بزنبرك متدرّج.
- **`.undo-win`:** كبسولة سوداء عائمة + أيقونة دائرية + رسالة + عدّاد + زر تراجع + شريط نزيف `5s`.

## 15. الستيكر ذاتي الرسم (`.fd-hero__stk` — نمط GLASS × SELF-DRAW)
- الأيقونة تُرسم نفسها عبر `clip-path` عند الدخول، ثم يعبر لمعان قطري كمصادقة على الاكتمال. قبّة زجاج علوية. **لمسة ساعات فاخرة.**

## 16. التذييل التوقيعي (`.f-sigfoot`)
- فاصل بمعيّن ذهبي مركزي + سطر العلامة + credit ثنائي اللغة (عربي Alexandria + لاتيني **Playfair Display مائل**) بأقواس زاوية ذهبية. طباعة راقية متحفّظة.

## أنماط الحالات الناقصة (مطلوب الالتزام بها في أي شاشة جديدة)

> هذه الأنماط **يجب** أن تُصمَّم لكل شاشة بيانات جديدة (معيار output-quality-rubrics):

- **الحالة الفارغة (Empty):** على غرار `.f-rem--add` — حدّ متقطّع، نبرة مهذّبة، دعوة واضحة أو «قريباً».
- **حالة التحميل (Loading):** هيكل عظمي (skeleton) بنفس أنصاف الأقطار والتباعد، لا spinner عاري.
- **حالة الخطأ (Error):** بلون `--f-err`، رسالة بشرية + إجراء استرداد، لا كود خام.
- **الحالة الحيّة (Live):** نقطة نبض (`f-pulse`) للمزامنة/الطوارئ.
