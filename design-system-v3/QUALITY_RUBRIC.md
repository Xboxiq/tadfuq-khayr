# معيار الجودة — Design System v3 (output-quality-rubrics)

> تعريف قابل للقياس لـ«ما هو جيّد» لأي شاشة/مكوّن قبل التسليم.
> كل بند يُقيَّم: ✅ مستوفٍ · 🟡 جزئي · ❌ غير مستوفٍ. لا تُسلَّم شاشة بأي ❌.

---

## 1. الدقة (Accuracy)
| البند | المعيار |
|------|---------|
| بيانات حقيقية | مربوطة بـ`data.js`، لا قيم وهمية مكتوبة يدوياً |
| لا أرقام مفبركة | المبالغ غير المعروفة تُعرض كـ«تُحدّد بعد الكشف»، لا رقم مخترع |
| مصدر القيمة | القيم الحسّاسة تحمل مصدراً (`v3-source`) أو حالة ثقة (`v3-trust`) |
| العملة والتنسيق | `fmtIQD` / `ar-IQ` لكل المبالغ، أرقام جدولية |

## 2. الملاءمة (Relevance)
| البند | المعيار |
|------|---------|
| تسلسل صحيح | الأهم بصرياً = الأهم وظيفياً (KPI اليوم، البلاغ العاجل) |
| لا ضجيج | كل عنصر له غرض؛ لا زخرفة بلا معنى |
| سياق محلّي | مصطلحات الكهرباء العراقية، RTL، أسماء واقعية |

## 3. الفائدة (Helpfulness)
| البند | المعيار |
|------|---------|
| إجراءات واضحة | لكل حالة إجراء تالٍ واضح (إشعار الطوارئ، عرض المزيد) |
| كشف تدريجي | التفاصيل عند الطلب لا دفعة واحدة (بحث متحوّل، أقسام تتوسّع) |
| من يقود؟ | تمييز «النظام يقود» (حفظ تلقائي) عن «المستخدم يقود» (تأكيد حرج) |

## 4. الثقة والشفافية (Trust + Transparency)
| البند | المعيار |
|------|---------|
| حالة الثقة ظاهرة | مؤكّد/تقديري/قيد التحديد عبر `v3-trust` |
| حدود الصلاحية | «يتطلب موافقة مدير» عبر `v3-gate` حيث ينطبق |
| نضارة البيانات | مؤشّر مزامنة (`v3-sync`) ومتى آخر تحديث |
| لا ادّعاء يقين زائف | لا نعرض رقماً نهائياً قبل الكشف الفعلي |

## 5. إمكانية الوصول (Accessibility)
| البند | المعيار |
|------|---------|
| تباين | نص/أزرار/نماذج تحقّق AA (4.5:1) في الوضعين |
| أهداف لمس | ≥ 44px لكل قابل للنقر |
| تركيز لوحة المفاتيح | `:focus-visible` ring واضح + `tabIndex` على الصفوف |
| دلالات | `role`, `aria-selected`, `aria-checked`, `aria-expanded` |

## 6. الحركة (Motion)
| البند | المعيار |
|------|---------|
| مبرَّرة | كل حركة لها سبب (تسلسل/تغذية راجعة/انتقال) |
| موحّدة | `--v3-ease`/`--v3-spring` فقط، `transform`/`opacity` فقط |
| reduced-motion | مسار كامل في `motion.css` لكل أنيميشن |

## 7. المتانة (Robustness)
| البند | المعيار |
|------|---------|
| الحالات الأربع | فارغ · تحميل (skeleton) · خطأ · مباشر — كلها مصمّمة |
| الاستجابة | bento ينهار بسلاسة (12 ← 6 ← 1 عمود) |
| الوضعان | نهاري + ليلي مُختبَران |
| عزل | `--v3-*` فقط، صفر تعديل على ملفات الأصل |

## 8. التحرير (Copy)
| البند | المعيار |
|------|---------|
| صفر em-dash | لا em-dash (—) في أي نص ظاهر |
| نبرة | رسمية-إنسانية، واضحة، لا حشو تسويقي |

## قائمة فحص سريعة قبل التسليم (Definition of Done)
- [ ] بيانات حقيقية من `data.js`، صفر أرقام مفبركة
- [ ] حالات الثقة/الصلاحية ظاهرة حيث تنطبق
- [ ] كشف تدريجي + تمييز قيادة النظام/المستخدم
- [ ] تباين AA · لمس ≥44px · focus ring · ARIA
- [ ] حركة مبرَّرة + reduced-motion
- [ ] فارغ/تحميل/خطأ/مباشر
- [ ] نهاري + ليلي
- [ ] صفر em-dash
- [ ] صفر تعديل على ملفات الأصل (تحقّق git)

## تطبيق السكلز الثمانية (تتبّع)

|--------|-------------------|
| design-taste-frontend | منع slop، bento مدروس، أقفال لون/شكل، صفر em-dash |

| trust-calibration | «تُحدّد بعد الكشف»، «يتطلب موافقة مدير»، حالات ثقة |
| transparency-patterns | مؤشّر مزامنة، مصدر البيانات، نضارة، حالة الحفظ |
| output-quality-rubrics | هذا الملف + قائمة الفحص |
